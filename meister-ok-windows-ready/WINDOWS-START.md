# Запуск Meister OK на Windows

## Что потребуется

1. Windows 10 или Windows 11 с поддержкой WSL 2.
2. Docker Desktop: https://www.docker.com/products/docker-desktop/
3. Не менее 4 ГБ свободной оперативной памяти и около 3 ГБ на диске для Docker-образов.

Node.js, pnpm, Prisma и SQLite отдельно устанавливать не нужно.

## Первый запуск

1. Распакуйте ZIP проекта в обычную папку, например `C:\Sites\meister-ok`.
2. Установите Docker Desktop, соглашаясь на использование WSL 2.
3. Перезагрузите Windows, запустите Docker Desktop и дождитесь сообщения, что Docker готов.
4. Дважды щёлкните `start.bat`.
5. При первом запуске скрипт автоматически:
   - создаст безопасный секрет;
   - создаст пароль администратора;
   - сохранит настройки в локальном `.env`;
   - соберёт и запустит сайт, базу и резервное копирование;
   - дождётся готовности базы;
   - откроет сайт в браузере.
6. Запишите показанный пароль администратора. Логин по умолчанию: `admin@meister-ok.local`.

Сайт: http://localhost:3000  
Админ-панель: http://localhost:3000/admin

Первый запуск может занять несколько минут, поскольку Docker загружает необходимые компоненты.

## Остановка

Дважды щёлкните `stop.bat`. Контейнеры остановятся, но база, фотографии и резервные копии сохранятся.

Для повторного запуска снова используйте `start.bat`.

## Где хранятся данные

Docker создаёт постоянные тома:

- база SQLite — `meister_data`;
- фотографии — `meister_uploads`;
- резервные копии — `meister_backups`.

Обычная остановка и перезагрузка компьютера не удаляют данные. Никогда не выполняйте `docker compose down -v`, если не хотите полностью удалить базу, фотографии и копии.

## Изменение логина или пароля

Откройте скрытый файл `.env` в Блокноте и измените `ADMIN_EMAIL` или `ADMIN_PASSWORD`. Пароль должен содержать минимум 12 символов. Затем запустите `stop.bat`, а после него `start.bat`.

## Если сайт не запускается

1. Убедитесь, что Docker Desktop запущен.
2. Откройте PowerShell в папке проекта.
3. Посмотрите состояние:

   ```powershell
   docker compose ps
   ```

4. Посмотрите журнал приложения:

   ```powershell
   docker compose logs app
   ```

5. Проверьте, не занят ли порт 3000. При необходимости измените `APP_PORT=3000` в `.env`, например на `APP_PORT=8080`. После этого сайт будет доступен по адресу `http://localhost:8080`.

`start.bat` автоматически прочитает новый `APP_PORT`, проверит и откроет правильный адрес.

## Публикация в интернете

Для открытого доступа перенесите проект на сервер с Linux и Docker, настройте домен и HTTPS через Caddy или Nginx. Полная инструкция находится в `README.md`.
