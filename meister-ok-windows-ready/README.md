# Meister OK

Готовый к размещению сайт веломагазина и мастерской на Next.js, TypeScript, Prisma и SQLite. В проект входят каталог, карточки товаров с фотогалереей, заявки в мастерскую, приватная переписка клиента с мастером и защищённая административная панель.

## Что уже настроено

- production-сборка Next.js;
- SQLite в постоянном Docker-томе;
- автоматические миграции при запуске;
- создание администратора из переменных окружения;
- JWT-сессия в защищённой `HttpOnly` cookie;
- загрузка до 8 фотографий JPG, PNG или WebP размером до 8 МБ;
- проверка MIME-типа и сигнатуры файла;
- постоянный Docker-том для фотографий;
- автоматическое резервное копирование базы и фотографий;
- хранение резервных копий с настраиваемым сроком;
- healthcheck `/api/health`;
- автоматический перезапуск контейнеров;
- базовые HTTP-заголовки безопасности;
- запуск приложения не от root-пользователя.

## Требования

На сервере нужны только:

- Docker Engine 24 или новее;
- Docker Compose v2;
- домен и reverse proxy с HTTPS для публичного размещения.

Node.js, pnpm, Prisma и SQLite на сервер отдельно устанавливать не требуется.

## Первый запуск через Docker

1. Скопируйте проект на сервер и перейдите в его каталог.

2. Создайте production-настройки:

   ```bash
   cp .env.production.example .env
   ```

3. Сгенерируйте секрет сессий:

   ```bash
   openssl rand -base64 48
   ```

4. Откройте `.env` и обязательно замените:

   - `AUTH_SECRET` — результат предыдущей команды, минимум 32 символа;
   - `ADMIN_EMAIL` — email администратора;
   - `ADMIN_PASSWORD` — уникальный пароль длиной минимум 12 символов;
   - `APP_PORT` — внешний порт, если стандартный `3000` не подходит.

5. Соберите и запустите:

   ```bash
   docker compose up -d --build
   ```

6. Проверьте состояние:

   ```bash
   docker compose ps
   docker compose logs -f app
   curl http://127.0.0.1:3000/api/health
   ```

Сайт доступен по `http://SERVER_IP:3000`, административная панель — по `/admin`.

При первом старте контейнер автоматически применит миграции, создаст SQLite-базу, базовые настройки и учётную запись администратора. Каталог останется пустым.

## Самый простой запуск на Windows

1. Установите и запустите Docker Desktop.
2. Распакуйте проект в папку без системных ограничений.
3. Дважды щёлкните `start.bat`.

Скрипт сам создаст `.env`, сгенерирует пароль администратора, выполнит `docker compose up -d --build`, дождётся готовности базы и откроет сайт. Для остановки используйте `stop.bat`; база, фотографии и резервные копии сохранятся.

Подробная инструкция для новичка находится в `WINDOWS-START.md`.

## Данные и Docker-тома

Compose создаёт три постоянных тома:

- `meister_data` — база `/data/meister.db`;
- `meister_uploads` — фотографии товаров;
- `meister_backups` — резервные копии базы и фотографий.

Команда `docker compose down` не удаляет данные. Не используйте `docker compose down -v`, если тома не были предварительно сохранены.

## Резервное копирование

Сервис `backup` создаёт первую копию после запуска приложения, затем повторяет операцию каждые `BACKUP_INTERVAL_HOURS` часов. Старые копии удаляются через `BACKUP_RETENTION_DAYS` дней.

Проверить журнал и список копий:

```bash
docker compose logs backup
docker compose exec backup ls -lah /backups
```

Создать дополнительную копию вручную:

```bash
docker compose exec backup node scripts/backup.mjs --once
```

Выгрузить копии с сервера в текущую папку:

```bash
docker compose cp backup:/backups ./backups-export
```

Рекомендуется дополнительно копировать `backups-export` во внешнее хранилище, которое не находится на том же сервере.

## Восстановление

1. Посмотрите имя нужной копии:

   ```bash
   docker compose exec backup ls -1 /backups
   ```

2. Остановите приложение и сервис копирования:

   ```bash
   docker compose stop app backup
   ```

3. Восстановите выбранную копию:

   ```bash
   BACKUP_NAME=2026-07-15T12-00-00-000Z docker compose --profile tools run --rm restore
   ```

4. Запустите сервисы снова:

   ```bash
   docker compose up -d
   ```

## Обновление проекта

```bash
docker compose exec backup node scripts/backup.mjs --once
git pull
docker compose up -d --build
docker image prune -f
```

Новые миграции применяются автоматически до запуска приложения.

## Настройка HTTPS

Для публикации не открывайте порт приложения напрямую в интернет. Разместите перед ним Caddy, Traefik или Nginx и проксируйте домен на `127.0.0.1:3000` с HTTPS.

Пример для Caddy:

```caddyfile
meister-ok.example.com {
    reverse_proxy 127.0.0.1:3000
    encode zstd gzip
}
```

После подключения домена ограничьте публикацию порта в `compose.yaml`:

```yaml
ports:
  - "127.0.0.1:3000:3000"
```

## Административная панель

Откройте `/admin` и войдите с `ADMIN_EMAIL` и `ADMIN_PASSWORD` из `.env`. В панели можно:

- добавлять и публиковать велосипеды;
- прикреплять несколько фотографий;
- удалять товары вместе с их фотографиями;
- просматривать заявки и отвечать клиентам;
- изменять телефон, WhatsApp, email, адрес, график и текст главной страницы.

Изменение `ADMIN_PASSWORD` в `.env` и последующий `docker compose up -d` обновляет пароль администратора.

## Структура

```text
app/                    страницы, Server Actions и healthcheck
components/             интерфейс и фотогалерея
lib/                    база, настройки и авторизация
prisma/                 схема, миграции и seed
public/uploads/         фотографии вне Docker
scripts/backup.mjs      резервное копирование
scripts/restore.sh      восстановление
Dockerfile              production-образ
compose.yaml            приложение, backup и restore
```

## Важные замечания перед публикацией

- никогда не добавляйте `.env` в Git;
- используйте уникальный пароль администратора;
- включите HTTPS до передачи реальных клиентских данных;
- регулярно проверяйте возможность восстановления резервной копии;
- ограничьте SSH и открытые порты серверным firewall;
- обновляйте Docker-образ и зависимости по расписанию.
